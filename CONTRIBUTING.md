# Contributing

Keep changes scoped, preserve existing behavior unless the change is documented,
and add tests proportional to risk. Run the checks in
[docs/DEVELOPMENT.md](docs/DEVELOPMENT.md) before submitting a pull request.

Do not edit released migrations or generated `web/` files by hand. Change
`frontend/src/`, run `npm run build`, and commit source plus output. Identify the
origin and license of any copied code, text, icon, or asset. Contributions are
accepted under GPL-3.0-or-later.

Security reports belong in the private process described in
[SECURITY.md](SECURITY.md), not a public issue.
